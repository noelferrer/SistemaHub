# SistemaHub Brand Guidelines

> This document defines how the SistemaHub brand looks, sounds, and feels.
> Keep this up to date as the brand evolves.

---

## 1. Brand Identity

**Company Name:** SistemaHub
**Tagline:** Software built for real work.
**Mission:** We build practical, purpose-built web applications that simplify how businesses operate — tools that fit how you actually work, not the other way around.
**Website:** sistemahub.com
**Email:** hello@sistemahub.com

---

## 2. Logo

See `logo/option-b/` for all active logo files.

**Active option:** Option B — dual concentric arc orbital mark with Space Grotesk wordmark.

| File | Use |
|------|-----|
| `logo-primary.svg` | Default — on light or white backgrounds |
| `logo-dark.svg` | On dark backgrounds (website nav, dark sections) |
| `logo-icon.svg` | Icon only — favicons, small spaces, app icons |

**Wordmark:** "Sistema" in Space Grotesk Regular · "Hub" in Space Grotesk Bold, Brand Orange (`#f39f5a`)

**Clear space rule:** Maintain padding equal to the icon height on all sides.

**Do not:**
- Recreate or redraw the logo
- Change the font or weights
- Place the logo on cluttered or low-contrast backgrounds
- Recolor the logo outside the approved variants

## 3. Color Palette

### Primary Palette

| Name | Hex | Usage |
|------|-----|-------|
| Brand Dark | `#0B0C37` | Background, dark sections, nav |
| Brand Orange | `#f39f5a` | Primary accent, CTAs, buttons, highlights, "Hub" wordmark |
| Brand Rose | `#ae445a` | Secondary accent, links, gradient start |

**Rose → Orange** · `linear-gradient(135deg, #ae445a 0%, #f39f5a 100%)`
Used for: icon marks, hero accents, gradient text, illustrations.

## 4. Typography

| Role | Font | Weight(s) | Usage |
|------|------|-----------|-------|
| Display / Heading | Space Grotesk | 700 (Bold) | Page titles, hero headlines, section headers |
| Subheading | Space Grotesk | 600 (Semibold) | Card titles, subsections, product names |
| Body | Inter | 400 / 500 | Paragraphs, descriptions, UI text |
| Mono | JetBrains Mono | 400 / 500 | Code, labels, badges, eyebrow text |

**Letter spacing:** Display headings use `-0.04em` tracking (tight). Mono labels use `+0.14em` tracking (wide uppercase).

**Line height:** Body text `1.7–1.8`. Headlines `1.0–1.05`.

## 5. Tone of Voice

**We are:**
- Professional but approachable
- Clear and direct — no jargon
- Confident without being arrogant
- Honest about what the software does and doesn't do

**We are not:**
- Stiff or overly corporate
- Overly casual or informal
- Vague or filled with buzzwords
- Hyperbolic ("revolutionary", "game-changing")

**Examples:**
- ✅ "We build tools that make running a business easier."
- ✅ "No sales calls, no fluff."
- ✅ "Software that fits how you operate."
- ❌ "We leverage cutting-edge synergistic solutions."
- ❌ "Disrupting the enterprise software space."

---

## 6. Imagery Style

- **Aesthetic:** Clean, dark, minimal. Avoid stock photos of people in offices.
- **Icons:** Outline style, 1.5px stroke, rounded caps and joins. No fill icons.
- **Illustrations:** Geometric, abstract, brand-colored. Use the gradient sparingly.
- **Backgrounds:** Dark navy-purple (`#0B0C37`) with subtle dot grids and radial glows.
- **Decorative elements:** Orbital arc marks (from the logo motif) used large and very faint as background texture.

---

## 7. Do's and Don'ts

**Do:**
- Use official logo files from `brand/logo/option-b/`
- Use the defined color palette for all brand materials
- Use Space Grotesk for all headings
- Maintain generous whitespace — the brand feels spacious, not dense

**Don't:**
- Recreate or modify the logo
- Use off-brand colors (no pure white text on orange backgrounds)
- Use a different font for headings
- Place the logo on cluttered or busy backgrounds
- Use the full gradient on body text
